## 2025-05-24 - [N+1 Query Optimization in Reports List]
**Learning:** In Laravel applications with multiple database connections, N+1 query regressions can easily occur when serializing models with appended attributes or when performing manual loops over collections for data aggregation. Using `groupBy` with `selectRaw` for counts and `whereExists` for complex filtering significantly improves performance from O(N) to O(1) queries. Additionally, testing dual-connection setups requires explicit configuration overrides in the `setUp` method to ensure both connections use in-memory SQLite during tests.
**Action:** Always prefer database-level aggregation over in-memory collection manipulation for large datasets and ensure test suites specifically monitor query counts for critical paths.
