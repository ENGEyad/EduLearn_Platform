## 2026-04-15 - [Dual Connection Testing with SQLite In-Memory]
**Learning:** In a Laravel application with multiple database connections (e.g., 'mysql' and 'app_mysql'), using `RefreshDatabase` for testing with SQLite in-memory requires special handling. Even if `DB_CONNECTION` is set to `sqlite`, other connections might still attempt to use their default drivers (like 'mysql'), leading to "Connection refused" errors during migrations.
**Action:** (1) Update `config/database.php` to allow `driver` injection via env variables (e.g., `DB_DRIVER`, `APP_DB_DRIVER`). (2) In the test `setUp()`, override connection configs. (3) Override `refreshTestDatabase()` in the test class to manually run `migrate:fresh` and reset the Artisan kernel to ensure all connections are migrated correctly for each test.
